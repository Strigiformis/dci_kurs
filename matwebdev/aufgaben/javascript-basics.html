I. Die Dateitypen in JavaScript

In JavaScript gibt es sieben verschiedene Datentypen. Ein Datentyp bezeichnet eine bestimmte Menge gleichartiger Werte und legt fest, welche Operationen man mit ihnen ausführen kann. Ein Beispiel dafür ist die Menge aller Zahlen, die JavaScript verarbeiten kann, und mathematischen Operationen wie Addition oder Multiplikation.

Ein weiterer Typ wären die Zeichenketten, zum Beispiel "Hallo" und "Welt", mit der Operation Verkettung: "Hallo" + "Welt" ergibt "HalloWelt"


Einfache Dateitypen

  • Undefined, keine Wert zugewiesen=wird in JavaScript verwendet um „kein Wert vorhanden“ darzustellen
  • Null
  • Boolean, kann den Wert true oder false haben
  • String, Zeichenkette
  • Symbol, zum Erstellen eines Zugriffschlüssels
  • Number, Zahl
  • BigInt, ganzzahliger Wert

Dateitypen müssen vor Variablen definiert werden!


Spezielle Dateitypen

  • Function, ist eine Folge von Anweisungen. Außer die Funktion aufzurufen, kann man einem Funktionsobjekt auch Eigenschaften zuweisen, man kann es in einer Variablen speichern oder als Parameter an andere Funktionen weitergeben. Die Klammern, die in JavaScript einen Funktionsaufruf kennzeichnen, werden Operator genannt.
  • Array

Was ist der Unterschied zwischen " == " und " === "?


"==" ist ein Vergleichsoperator, liefert einen booleschen Wert. Bei gleichen Werten ist das Ergebnis der Operation "true", ansonsten "false". Vergleicht man allerdings zwei unterschiedliche Typen, versucht der Operator eine Typumwandlung durchzuführen.

Deshalb sollte man "===" verwenden da bei einem solchen Vergleich immer die Operation abgebrochen und der Wert "false" ausgegeben wird.
